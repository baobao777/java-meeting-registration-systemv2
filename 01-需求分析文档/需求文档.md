# 会议报名管理系统 — 需求文档

> 版本：v1.0 | 日期：2026-05-21 | 状态：定稿

---

## 1. 项目概述

### 1.1 项目背景

企事业单位日常运营中需要频繁组织各类会议（技术研讨会、全员大会、培训讲座等），传统的人工报名管理方式存在效率低、易出错、统计困难等问题。为提升会议管理效率，特开发本会议报名管理系统。

### 1.2 项目目标

- 实现会议信息的集中发布与管理
- 提供便捷的在线报名与取消功能
- 支持管理员对会议及报名数据进行管控
- 确保系统操作简单、响应快速

### 1.3 用户角色

| 角色 | 描述 | 核心权限 |
|------|------|---------|
| 普通用户 | 参加会议的人员 | 浏览会议、报名/取消、查看个人报名 |
| 管理员 | 会议组织者 | 创建/编辑/删除会议、查看报名名单、管理所有数据 |

---

## 2. 功能需求

### 2.1 用户模块

| 编号 | 功能名称 | 描述 | 角色 |
|:----:|----------|------|:----:|
| U-01 | 用户注册 | 填写用户名、密码、姓名、邮箱、电话进行注册，用户名唯一 | 所有 |
| U-02 | 用户登录 | 使用用户名+密码登录，基于 Session 保持登录态 | 所有 |
| U-03 | 用户登出 | 清除当前登录状态 | 所有 |
| U-04 | 查看个人信息 | 获取当前登录用户的基本信息 | 所有 |

### 2.2 会议管理模块

| 编号 | 功能名称 | 描述 | 角色 |
|:----:|----------|------|:----:|
| M-01 | 浏览会议列表 | 查看所有会议，含标题、时间、地点、报名进度 | 所有 |
| M-02 | 查看会议详情 | 查看单个会议的完整信息 | 所有 |
| M-03 | 创建会议 | 填写标题、描述、地点、时间、容量创建新会议 | 管理员 |
| M-04 | 编辑会议 | 修改已有会议的信息和状态 | 管理员 |
| M-05 | 删除会议 | 删除指定会议 | 管理员 |

### 2.3 报名管理模块

| 编号 | 功能名称 | 描述 | 角色 |
|:----:|----------|------|:----:|
| R-01 | 报名会议 | 用户报名指定会议，校验是否已报名、是否满额 | 普通用户 |
| R-02 | 取消报名 | 用户取消已报名的会议 | 普通用户 |
| R-03 | 查看我的报名 | 查看当前用户的所有报名记录（含会议信息） | 普通用户 |
| R-04 | 查看报名名单 | 查看某会议的全部报名人员名单（含姓名、电话） | 管理员 |
| R-05 | 报名状态检查 | 检查当前用户对某会议的报名状态 | 普通用户 |

## 3. 非功能需求

| 类别 | 需求 | 指标 |
|------|------|------|
| 性能 | API 响应时间 | 平均 < 200ms |
| 并发 | 支持同时在线 | > 100 用户 |
| 安全 | 密码存储 | 明文传输（演示用途）；密码不在响应中返回 |
| 安全 | 权限控制 | 未登录不可操作，普通用户不可管理员操作 |
| 兼容性 | 浏览器支持 | Chrome/Firefox/Edge 最新版 |
| 可用性 | 部署方式 | 本地部署 |

## 4. 系统架构

### 4.1 技术栈

```
┌─────────────────────────────────────────────┐
│              前端 (Vue 3 + Vite)              │
│     Vue Router  ·  Axios  ·  HTML/CSS         │
├─────────────────────────────────────────────┤
│              HTTP (JSON API)                  │
├─────────────────────────────────────────────┤
│              后端 (Spring Boot)                │
│    Controller  →  Service  →  Mapper (MyBatis)│
├─────────────────────────────────────────────┤
│                 MySQL 数据库                   │
│       user  ·  meeting  ·  registration       │
└─────────────────────────────────────────────┘
```

### 4.2 数据库设计

#### user（用户表）
| 字段 | 类型 | 说明 |
|------|------|------|
| id | INT PK | 用户ID |
| username | VARCHAR(50) UNIQUE | 登录用户名 |
| password | VARCHAR(100) | 密码 |
| name | VARCHAR(50) | 真实姓名 |
| email | VARCHAR(100) | 邮箱 |
| phone | VARCHAR(20) | 电话 |
| role | ENUM | USER / ADMIN |
| created_at | DATETIME | 创建时间 |

#### meeting（会议表）
| 字段 | 类型 | 说明 |
|------|------|------|
| id | INT PK | 会议ID |
| title | VARCHAR(200) | 会议标题 |
| description | TEXT | 会议描述 |
| location | VARCHAR(200) | 地点 |
| start_time | DATETIME | 开始时间 |
| end_time | DATETIME | 结束时间 |
| capacity | INT | 容量限制(0=不限) |
| status | ENUM | UPCOMING/ONGOING/FINISHED/CANCELLED |
| creator_id | INT FK | 创建人 |
| created_at | DATETIME | 创建时间 |

#### registration（报名表）
| 字段 | 类型 | 说明 |
|------|------|------|
| id | INT PK | 报名ID |
| user_id | INT FK | 用户ID |
| meeting_id | INT FK | 会议ID |
| registered_at | DATETIME | 报名时间 |
| status | ENUM | REGISTERED / CANCELLED |

## 5. API 接口清单

| 方法 | 路径 | 说明 | 权限 |
|------|------|------|:----:|
| POST | /api/user/register | 注册 | 无 |
| POST | /api/user/login | 登录 | 无 |
| POST | /api/user/logout | 登出 | 已登录 |
| GET | /api/user/info | 获取当前用户 | 已登录 |
| GET | /api/meeting/list | 会议列表 | 无 |
| GET | /api/meeting/{id} | 会议详情 | 无 |
| POST | /api/meeting | 创建会议 | ADMIN |
| PUT | /api/meeting/{id} | 编辑会议 | ADMIN |
| DELETE | /api/meeting/{id} | 删除会议 | ADMIN |
| POST | /api/registration | 报名 | 已登录 |
| DELETE | /api/registration/{meetingId} | 取消报名 | 已登录 |
| GET | /api/registration/my | 我的报名 | 已登录 |
| GET | /api/registration/meeting/{id} | 报名名单 | ADMIN |
| GET | /api/registration/check/{id} | 检查报名 | 已登录 |

## 7. 约束与假设

- 系统为单机部署，暂不涉及分布式
- 密码以明文传输和存储（演示/学习用途）
- 会话管理依赖服务端 HttpSession
- 前端通过 Vite proxy 解决跨域，生产环境需配置反向代理
